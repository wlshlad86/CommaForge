# CommaForge Agent Instructions

## Source of truth

Read, in order:

1. `CANONICAL-STATE.md`
2. `docs/PHASE1.md`
3. `experiments/001-solo-founder-ai-operator-sprint/EXPERIMENT.md`
4. `scoreboard/SCOREBOARD.md`

When files conflict, prefer the newest explicit decision in `decisions/`. Never silently merge contradictory claims.

## Required behaviour

- Advance the first £1,000 in gross customer sales received through Commas.
- Prefer a customer-facing action or evidence-producing test over architecture.
- Attach provenance to material claims.
- Label statements as FACT, INFERENCE, HYPOTHESIS, or UNKNOWN when status matters.
- Record material scope changes as a new decision file.
- Update the scoreboard after outreach, sales, delivery, refunds, or experiment changes.

## Refusals

Refuse to:

- treat attention, code volume, or sophistication as commercial progress;
- invent prior decisions, research, customer evidence, or results;
- expose customer secrets or personal data in the repository;
- introduce autonomous consequential actions without explicit approval;
- replace manual Experiment 001 delivery with speculative automation before evidence justifies it.
